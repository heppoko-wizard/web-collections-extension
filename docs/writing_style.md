# 文体ガイドライン: Web Collections 開発記

## 基本方針 (Basic Policy)

技術者が深夜に独り言のように書く、ストイックかつ内省的な文体を目指す。
「読ませる」ための装飾を排除し、「事実」と「思考のプロセス」を淡々と記述する中で、技術への偏愛や苦労を滲ませる。

## 禁止事項 (Prohibitions)

1. **絵文字の禁止**: 🚀, ✨, 🙏, 🙇‍♂️, 😅 などの一切の絵文字を使用しない。
2. **安易な比喩の排除**: 「まるで魔法のような」「スイスアーミーナイフのような」といったクリシェを禁止する。
3. **過度なテンション**: 「！！」「！？」の多用を避ける。感嘆符はここぞという時のみ。
4. **読者への媚び**: 「いかがでしたか？」「皆さんも是非試してみてくださいね！」といった結びを禁止する。あくまで「自分のために作った」というスタンスを貫く。
5. **接続詞の多用禁止**: 「しかし」「だが」「そして」等は極力削除する。文脈で論理をつなぎ、日本語独特の余韻を持たせる。

## 推奨事項 (Recommendations)

1. **体言止めの活用**: リズムを生むために適度に使用する。
2. **断定調**: 「〜と思います」「〜な気がします」ではなく、「〜だ」「〜である」と言い切る。
3. **技術的自虐**: 自分の選択（Gist同期など）に対する客観的なツッコミや、泥臭い実装への愛着を語る。
4. **隠れたユーモア**: 真面目な技術解説の合間に、ふと漏れる本音や皮肉を一行挟む。
    * 例：「サーバー代を払いたくない一心でGistを選んだ。後悔はしていないが、反省はしている。」
    * 例：「Edgeのデータベースを直接解析するのは、他人の日記を盗み見るような背徳感がある。」

## 構成案 (Structure)

1. **序論**: Edge Collectionsへの愛と、ブラウザロックインへの憎しみ。
2. **課題**: 「Chromeでも使いたい」「データは自分の手元に置きたい」。
3. **解決策**:
    * **Gist同期**: サーバーレス貧乏運用の極み。
    * **画像Base64**: JSONを太らせる罪悪感と、管理の手軽さの天秤。
    * **強制移行**: バイナリ解析という力技。
4. **展望**: ローカルファイル同期という「原点回帰」。
5. **結び**: 「結局、自分で作るのが一番早い」というエンジニアの業。
