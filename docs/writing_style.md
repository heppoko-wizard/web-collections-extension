# 文体ガイドライン: Web Collections 開発記

## 基本方針 (Basic Policy)

技術者が深夜に独り言のように書く、ストイックかつ内省的な文体を目指す。
「読ませる」ための装飾を排除し、「事実」と「思考のプロセス」を淡々と記述する。

## 禁止事項 (Prohibitions)

1. **絵文字の禁止**: 🚀, ✨, 🙏, 🙇‍♂️, 😅 などの一切の絵文字を使用しない。
2. **安易な比喩の排除**: 「まるで魔法のような」「スイスアーミーナイフのような」といったクリシェを禁止する。
3. **AI的表現の禁止**: 「思考の断片」「可能性を秘めた」「〜の旅」「新たな地平」などの手垢のついた表現を排除する。
4. **ドラマチックな表現の禁止**: 「罪」「罠」「呪縛」「悪夢」「正気の沙汰ではない」など、感情を煽るような言葉を使用しない。
5. **過度なテンション**: 感嘆符（！）の使用を避ける。
6. **読者への媚び**: 「いかがでしたか？」などの結びを禁止する。
7. **接続詞の多用禁止**: 「しかし」「だが」等は極力削除する。
8. **カギ括弧の多用禁止**: 強調のための「」を避け、地の文で表現する。
9. **「〜という〜」表現の禁止**: 「クラウドストレージというローカル」のような、一見上手いことを言っている風な言い換えを禁止する。事実をそのまま記述する。

## 推奨事項 (Recommendations)

1. **具体性**: 抽象的な表現よりも、具体的な事実（ファイルサイズ、API仕様など）を書く。
2. **即物的な表現**: ことさらに感情を乗せず、事実関係を記述する。
3. **体言止めの活用**: リズムを生むために適度に使用する。
4. **断定調**: 「〜と思います」ではなく、「〜だ」「〜である」と言い切る。
5. **技術的自虐**: 自分の選択に対する客観的な分析や、妥協点についての率直な記述。
